# Packages

ExpressoTS v4 is split into a handful of focused npm packages. Most apps only consume **`@expressots/core`** and **`@expressots/adapter-express`** directly; the rest are tooling, optional integrations, or building blocks for advanced setups.

| Package | Role | When you import it |
| --- | --- | --- |
| **[`@expressots/core`](./core.mdx)** | DI container, lifecycle, validation, errors, providers, logger | Always |
| **[`@expressots/adapter-express`](./adapter-express.mdx)** | HTTP adapter — controllers, decorators, `bootstrap()`, `micro()` | Always (HTTP apps) |
| **[`@expressots/shared`](./shared.mdx)** | Shared types and utilities used across packages | Rarely (transitive) |
| **[`@expressots/cli`](./cli.mdx)** | `expressots new`, `expressots create`, `expressots studio` | Dev / CI |
| **[`@expressots/studio`](./studio.mdx)** | Studio UI (browser dashboard for inspecting apps) | Dev only |
| **[`@expressots/studio-agent`](./studio-agent.mdx)** | In-process agent that streams runtime info to Studio | Dev only |

All packages are released together under the same version line. For v4 previews that's `4.0.0-preview.N` on the `next` tag; the official 4.0.0 release will move to `latest`.

## What you actually import

A typical v4 app uses two imports:

```typescript
import {
    bootstrap,
    loadEnvSync,
    provide,
    inject,
    Logger,
    AppError,
} from "@expressots/core";

import {
    AppExpress,
    controller,
    Get,
    Post,
    body,
    param,
    validatedBody,
} from "@expressots/adapter-express";
```

Everything else (`@expressots/shared`, `@expressots/studio`, …) is either transitively pulled in by the two above, or is a `devDependency` you add when scaffolding (the CLI does this for you).

## Versioning policy

- **One version line** — every package in the monorepo ships the same version.
- **Conventional commits** drive the version bump (`feat` → minor, `fix` → patch, `feat!` / `BREAKING CHANGE` → major).
- Preview releases use the `next` npm tag: `npm install @expressots/cli@next`.
- Stable releases use the `latest` tag: `npm install @expressots/cli`.

## Source layout

The framework lives across several Git repositories — `expressots/expressots`, `expressots/adapter-express`, `expressots/shared`, `expressots/expressots-cli`, `expressots/expressots-studio`, and `expressots/templates`. They're versioned together but published independently to keep release blast radius small.

You don't need to know any of this to use ExpressoTS — but if you're filing an issue or sending a PR, this is the page that tells you which repo to head for.
