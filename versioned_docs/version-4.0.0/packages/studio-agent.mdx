# `@expressots/studio-agent`

The Studio agent runs **inside your application** during development. It collects:

- The route table (post-controller-discovery)
- The provider registry (with capabilities — `IBootstrap`, `IShutdown`, `IHealthCheck`, `IMetrics`)
- The middleware pipeline (resolved order, including built-ins from presets)
- The lifecycle hook registry
- Event handler registrations and fire counts
- Logger output (when subscribed)

…and exposes them on a WebSocket that the [Studio UI](./studio.mdx) connects to.

## How it gets installed

The agent is wired in for you whenever you scaffold from any v4 template:

- `application` — auto-mounted by `AppExpress.bootstrap()` when `NODE_ENV !== "production"`.
- `application-with-events` — same as above; events show up in Studio's event panel.
- `micro` — auto-mounted by `micro()` when not in production.
- `provider` — N/A (libraries don't run a server).

You can also enable it explicitly:

```typescript title="src/app.ts"
import { AppExpress } from "@expressots/adapter-express";
import { setupExpressoTSStudioAgent } from "@expressots/adapter-express";

class App extends AppExpress {
    async configureServices() {
        this.Middleware.applyPreset("api");
    }

    async postServerInitialization() {
        // Optional: explicit setup for non-default ports / paths
        await setupExpressoTSStudioAgent(this, {
            port: 8585,                    // default
            host: "127.0.0.1",             // never bind to 0.0.0.0
            enabled: process.env.NODE_ENV !== "production",
        });
    }
}
```

## What it streams

| Channel | Payload |
| --- | --- |
| `runtime/routes` | Detected controllers + methods (path, verb, version, guards, interceptors) |
| `runtime/providers` | Provider registry with capabilities and last health snapshot |
| `runtime/middleware` | Pipeline entries with category, type, order, and (optional) profiler stats |
| `runtime/lifecycle` | Bootstrap / shutdown hook execution status |
| `runtime/events` | Registered handlers, fire counts, last payload (when serializable) |
| `runtime/logs` | Live `Logger` stream, scoped per service |

The agent never sends application-state data (database rows, request bodies, secrets). It's a metadata-only protocol.

## Security

The agent has three guard rails:

1. **Production opt-out** — `NODE_ENV === "production"` disables the agent unless `enabled: true` is forced.
2. **Localhost binding** — defaults to `127.0.0.1`, not `0.0.0.0`. Reset only with explicit `host: "0.0.0.0"`.
3. **No request capture** — the agent reads route metadata, not request payloads.

If you need to inspect a remote staging instance, tunnel the WebSocket through SSH; don't expose the agent on a public address.

## Versioning

Released alongside `@expressots/adapter-express`. The Studio UI and agent stay in lockstep — connecting an older Studio to a newer agent (or vice versa) may show partial data.
