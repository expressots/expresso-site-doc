# Release notes

For full details on the changes for each release, please refer to the [upgrade guide](./upgrade_guide.mdx).

## Versioning scheme

ExpressoTS follows the [Semantic Versioning](https://semver.org/) scheme. Major versions are reserved for breaking changes, minor versions for new features, and patch versions for bug fixes.
Major versions are released yearly, we strive to release on Q4 or Q1 of each year. Minor and patch versions are released as needed, usually on by-weekly or monthly basis.

The releases from now on will be done in bundles. This is to make it easier to keep track of the changes and to avoid having to release a new version for every small change.
The packages part of the bundle will be released at the same time, and the version number will be the same for all of them. This is to avoid any compatibility issues between the packages.

The packages part of the bundle will be:

-   `@expressots/core`
-   `@expressots/adapter-express`
-   `@expressots/shared`
-   `@expressots/templates`
-   `@expressots/cli`

## Support policy

For all ExpressoTS releases, bug fixes and security patches will be provided for 18 months after the release date. After that period, the release will be considered end-of-life and no further support will be provided except
for `Enterprise customers` or `individuals` with `valid support contracts`.

## ExpressoTS 4.0 🎉

ExpressoTS v4.0.0 is a major milestone in our journey toward providing a faster, more scalable, and developer-friendly Node.js framework.
With this release, we've implemented significant features, optimizations, and architectural changes to enhance your development experience.

Here's everything you need to know about what's new in ExpressoTS v4.0.0!

### Nodejs version

ExpressoTS v4.0.0 requires Node.js version 20.18.0 or higher.

### What's New in v4.0.0

ExpressoTS 4.0 introduces **5 major core features** that make it one of the most comprehensive TypeScript frameworks available:

#### 1. 🎯 Interceptors System

A complete AOP (Aspect-Oriented Programming) interceptor system for cross-cutting concerns:

-   **Conditional Interceptors**: `whenInterceptor()`, `unlessInterceptor()` for conditional execution
-   **Auto-Discovery**: Interceptors are automatically discovered via `@provide()` + `@UseInterceptors()`
-   **Composition Utilities**: `pipeInterceptors()`, `combineInterceptors()` for semantic grouping
-   **Built-in Interceptors**: `PerformanceInterceptor`, `LoggingInterceptor`, `TimeoutInterceptor`
-   **Type-Safe**: Full TypeScript inference on response transformations

```ts
import { UseInterceptors, PerformanceInterceptor, whenInterceptor } from "@expressots/core";

@controller("/users")
export class UserController {
    @Get("/")
    @UseInterceptors(
        PerformanceInterceptor,
        whenInterceptor(ctx => ctx.request.headers['x-cache'] === 'true', CacheInterceptor)
    )
    getUsers() {
        return this.userService.findAll();
    }
}
```

#### 2. 🧪 Testing Module

Comprehensive testing utilities for unit, integration, and E2E testing:

-   **Zero-Config Setup**: `createTestApp()` for instant test app creation
-   **Smart Mocks**: `mockProvider()` with auto-suggestion of dependencies
-   **Fluent HTTP Testing**: Chainable, readable API better than supertest
-   **Snapshot Testing**: API snapshot testing with `snapshotRequest()`
-   **Load Testing**: Built-in load testing utilities with percentile metrics
-   **Database Testing**: `createTestDatabase()` with fixtures support

```ts
import { createTestApp, request } from "@expressots/testing";

describe("UserController", () => {
    const { app, container } = await createTestApp(App);

    test("GET /users returns user list", async () => {
        await request(app)
            .get("/users")
            .expectStatus(200)
            .expectBody<User[]>(users => users.length > 0)
            .expectTime({ lessThan: 100 });
    });
});
```

#### 3. 📡 Event System

Type-safe, auto-discovered event system for decoupled architecture:

-   **Type-Safe Events**: Event classes instead of strings for full TypeScript inference
-   **Auto-Discovery**: Event handlers discovered via `@OnEvent()` decorator
-   **Priority Execution**: Multiple handlers with execution priority
-   **Conditional Handlers**: `@When()` decorator for conditional execution
-   **Event Replay**: Built-in event recording and replay for debugging
-   **Flow Visualization**: ASCII art visualization of event propagation

```ts
// Type-safe event class
export class UserCreatedEvent {
    constructor(
        public readonly userId: string,
        public readonly email: string
    ) {}
}

// Auto-discovered handler
@provide(SendWelcomeEmailHandler)
@OnEvent(UserCreatedEvent, { priority: 1 })
export class SendWelcomeEmailHandler implements IEventHandler<UserCreatedEvent> {
    handle(event: UserCreatedEvent) {
        // Full TypeScript inference on event payload
        this.emailService.sendWelcome(event.email);
    }
}
```

#### 4. ⏱️ Lazy-Loading Modules

Performance optimization for large applications:

-   **On-Demand Loading**: Modules loaded only when first accessed
-   **Auto-Detection**: Framework detects which modules can be lazy-loaded
-   **Preloading Hints**: Configure preloading for critical modules
-   **Progressive Loading**: Spread startup cost over time

```ts
import { LazyModule } from "@expressots/core";

// Module only loaded when /admin routes are accessed
@LazyModule({ preload: false })
export class AdminModule {
    // Heavy admin functionality loaded on demand
}
```

#### 5. ⚙️ Enhanced Configuration

Type-safe configuration with helpful validation:

-   **Type-Safe Config**: Full TypeScript inference on configuration
-   **Helpful Validation**: Detailed error messages with examples
-   **Environment Switching**: Automatic config switching based on environment
-   **Secret Management**: `SecretValue` class for secure handling

```ts
import { defineConfig, env } from "@expressots/core";

export default defineConfig({
    database: {
        url: env.string("DATABASE_URL").required(),
        pool: env.number("DB_POOL_SIZE").default(10),
    },
    features: {
        caching: env.boolean("ENABLE_CACHE").default(true),
    },
});
```

### Enhanced Middleware System

Building on v3's middleware system, v4 adds:

-   **Middleware Presets**: Pre-configured bundles (`api`, `production`, `development`)
-   **Performance Profiling**: Built-in middleware profiler with metrics
-   **Pipeline Introspection**: Query middleware pipeline at runtime
-   **Smart Validation**: Zero-config validation with `@validatedBody()`

### Enhanced Authorization System

Comprehensive guard system with:

-   **Built-in Guards**: `AuthenticatedGuard`, `RoleGuard`, `PermissionGuard`, `ResourceOwnerGuard`
-   **ABAC Support**: Attribute-Based Access Control with `RequirePolicy()`
-   **Guard Composition**: `combineGuards()`, `sequenceGuards()`, `whenGuard()`
-   **Guard Caching**: Request-scoped guard result caching
-   **Permission Hierarchy**: Role/permission inheritance support

### Enhanced Error Handling

RFC 7807 compliant error handling:

-   **Exception Filters**: Auto-discovered via `@Catch()` decorator
-   **Route-Specific Filters**: `@UseFilters()` at controller and method level
-   **Enhanced AppError**: Static helpers like `badRequest()`, `notFound()`, `validationFailed()`
-   **Problem Details**: RFC 7807 compliant error responses

### CLI Improvements

-   **Micro API Template**: Lightweight template for quick prototyping
-   **Plugin Management**: `ex add` and `ex remove` for packages
-   **Script Runner**: `ex scripts` for interactive script execution
-   **Improved Performance**: Using `tsx` for development and `tsc` for production

### Breaking Changes

-   Minimum Node.js version: 20.18.0
-   Updated dependency injection patterns
-   Enhanced lifecycle hooks in `app.ts`

For migration instructions, see the [upgrade guide](./upgrade_guide.mdx).
