# Testing Comparison

Compare ExpressoTS's testing utilities with other major frameworks.

## Testing Features

| Feature | ExpressoTS | Spring Boot | ASP.NET Core | NestJS | Django | Laravel |
|---------|-----------|-------------|--------------|--------|--------|---------|
| **Testing Module** | ✅ **PRODUCTION-READY** | ✅ `@SpringBootTest` | ✅ `TestHost` | ✅ `Test` module | ✅ Django test client | ✅ Laravel testing |
| **Zero-Config Setup** | ✅ **UNIQUE** | ⚠️ Requires annotation | ⚠️ Requires setup | ⚠️ Requires setup | ⚠️ Requires setup | ⚠️ Requires setup |
| **Smart Mocks** | ✅ **UNIQUE** - Auto-mock | ⚠️ Mockito | ⚠️ Moq | ⚠️ Jest manual | ⚠️ unittest.mock | ⚠️ Mockery |
| **HTTP Testing** | ✅ **Fluent API** | ✅ MockMvc | ✅ HttpClient | ✅ Supertest | ✅ Test client | ✅ Laravel testing |
| **API Snapshot Testing** | ✅ **Built-in** | ⚠️ Plugin | ⚠️ Plugin | ⚠️ Plugin | ⚠️ Plugin | ⚠️ Plugin |
| **Load Testing** | ✅ **Built-in** | ⚠️ JMeter | ⚠️ External tools | ⚠️ External tools | ⚠️ External tools | ⚠️ External tools |
| **Database Testing** | ✅ Built-in utilities | ✅ `@DataJpaTest` | ✅ InMemory | ✅ TypeORM testing | ✅ Django ORM | ✅ DatabaseMigrations |
| **Context Mocking** | ✅ **UNIQUE** - Request context | ⚠️ Manual | ⚠️ Manual | ⚠️ Manual | ⚠️ Manual | ⚠️ Manual |

### ExpressoTS Advantage

- ✅ **Zero-config setup** - Works out of the box
- ✅ **Smart mocks** - Automatic mock generation
- ✅ **Fluent HTTP testing** - Intuitive API for testing endpoints
- ✅ **Built-in load testing** - No external tools needed
- ✅ **API snapshot testing** - Capture and compare responses

### Zero-Config Setup

```typescript
import { TestingModule } from "@expressots/testing";

describe("UserController", () => {
    let testModule: TestingModule;

    beforeEach(async () => {
        // Zero config - just specify controllers
        testModule = await TestingModule.create({
            controllers: [UserController],
        });
    });

    afterEach(async () => {
        await testModule.close();
    });
});
```

### Smart Mocks

```typescript
// Automatic mock generation for services
testModule = await TestingModule.create({
    controllers: [UserController],
    providers: [
        {
            provide: UserService,
            useValue: TestingModule.autoMock(UserService),
        },
    ],
});

// Smart mock with automatic type inference
const mockUserService = testModule.get(UserService);
mockUserService.findById.mockResolvedValue({ id: "1", name: "Test" });
```

### Fluent HTTP Testing

```typescript
describe("User API", () => {
    it("should create user", async () => {
        const response = await testModule
            .request()
            .post("/users")
            .send({ name: "John", email: "john@test.com" })
            .expect(201)
            .expect("Content-Type", /json/);

        expect(response.body).toMatchObject({
            name: "John",
            email: "john@test.com",
        });
    });

    it("should return 404 for unknown user", async () => {
        await testModule
            .request()
            .get("/users/unknown-id")
            .expect(404);
    });
});
```

### API Snapshot Testing

```typescript
it("should match API snapshot", async () => {
    const response = await testModule
        .request()
        .get("/users/1");

    // Automatically captures and compares response
    expect(response.body).toMatchSnapshot();
});
```

### Built-in Load Testing

```typescript
import { LoadTester } from "@expressots/testing";

describe("Performance", () => {
    it("should handle load", async () => {
        const results = await LoadTester.run({
            target: testModule,
            endpoint: "/users",
            duration: 10, // seconds
            concurrency: 50,
        });

        expect(results.avgLatency).toBeLessThan(100); // ms
        expect(results.errorRate).toBeLessThan(0.01); // 1%
        expect(results.requestsPerSecond).toBeGreaterThan(100);
    });
});
```

### Database Testing

```typescript
import { DatabaseTestUtils } from "@expressots/testing";

describe("User Repository", () => {
    let dbUtils: DatabaseTestUtils;

    beforeEach(async () => {
        dbUtils = await DatabaseTestUtils.create({
            entities: [User],
            synchronize: true,
        });
    });

    afterEach(async () => {
        await dbUtils.cleanup();
    });

    it("should save user", async () => {
        const user = await dbUtils.repository(User).save({
            name: "Test User",
            email: "test@example.com",
        });

        expect(user.id).toBeDefined();
    });
});
```

### Request Context Mocking

```typescript
it("should authenticate user", async () => {
    const response = await testModule
        .request()
        .withContext({
            user: { id: "1", roles: ["admin"] },
            headers: { authorization: "Bearer token" },
        })
        .get("/admin/dashboard")
        .expect(200);
});
```
