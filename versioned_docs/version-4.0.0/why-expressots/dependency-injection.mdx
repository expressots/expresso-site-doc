# Dependency Injection Comparison

Compare ExpressoTS's DI system with other major frameworks.

## DI Container

| Feature | ExpressoTS | Spring Boot | ASP.NET Core | NestJS | Django | Laravel |
|---------|-----------|-------------|--------------|--------|--------|---------|
| **DI Container** | ✅ InversifyJS | ✅ Spring IoC | ✅ `IServiceProvider` | ✅ Built-in | ❌ Not native | ✅ Service Container |
| **Constructor Injection** | ✅ `@inject()` | ✅ Auto | ✅ Auto | ✅ Auto | ⚠️ Manual | ✅ Auto |
| **Property Injection** | ✅ Supported | ✅ `@Autowired` | ✅ `[FromServices]` | ✅ `@Inject()` | ❌ Not available | ✅ Via container |
| **Interface Binding** | ✅ Symbol/string IDs | ✅ Interface proxies | ✅ Interface registration | ✅ Interface tokens | ❌ Not available | ✅ Interface binding |
| **Factory Pattern** | ✅ `toFactory()` | ✅ `@Bean` | ✅ `AddSingleton(factory)` | ✅ `useFactory` | ⚠️ Manual | ✅ Factory binding |
| **Multi-injection** | ✅ `@multiInject()` | ✅ `List\<T\>` injection | ⚠️ Manual array | ✅ `@Inject()` array | ❌ Not available | ⚠️ Manual |
| **Optional Injection** | ✅ `@optional()` | ✅ `@Autowired(required=false)` | ✅ `GetService()` | ✅ `@Optional()` | ❌ Not available | ⚠️ Manual |
| **Named Bindings** | ✅ `@named()` | ✅ `@Qualifier` | ⚠️ Named services | ✅ Custom tokens | ❌ Not available | ✅ Named bindings |
| **Lazy Injection** | ✅ `@lazyInject()` | ✅ `@Lazy` | ✅ `Lazy\<T\>` | ✅ `Lazy\<T\>` | ❌ Not available | ⚠️ Manual |
| **Custom Scopes** | ✅ **UNIQUE** - `@provideInScope()` | ✅ Custom scopes | ⚠️ Custom `IServiceScope` | ❌ Not available | ❌ Not available | ⚠️ Manual |

## Service Scopes

| Feature | ExpressoTS | Spring Boot | ASP.NET Core | NestJS | Django | Laravel |
|---------|-----------|-------------|--------------|--------|--------|---------|
| **Singleton** | ✅ `provideSingleton()` | ✅ `@Singleton` (default) | ✅ `AddSingleton()` | ✅ `DEFAULT` | ⚠️ Module-level | ✅ Singleton binding |
| **Transient** | ✅ `provideTransient()` | ✅ `@Prototype` | ✅ `AddTransient()` | ✅ `TRANSIENT` | ⚠️ Function factories | ✅ Instance binding |
| **Request Scoped** | ✅ `inRequestScope()` | ✅ `@RequestScope` | ✅ `AddScoped()` | ✅ `REQUEST` | ⚠️ Manual | ✅ Request binding |
| **Custom Scopes** | ✅ **UNIQUE** - Full support | ✅ Custom scope impl | ⚠️ Custom `IServiceScope` | ❌ Not available | ❌ Not available | ⚠️ Manual |

### ExpressoTS Advantage

- ✅ **Most flexible DI system** in Node.js ecosystem
- ✅ **Custom scopes** - unique feature for tenant isolation, transaction management, etc.
- ✅ **InversifyJS power** - most advanced DI container in TypeScript

### Custom Scopes Example

ExpressoTS supports custom scopes for advanced scenarios like tenant isolation:

```typescript
// Using decorator
@provideInScope(TenantService, "tenant")
class TenantConfigService {}

// Using direct binding
container.bind(ServiceIdentifier).to(ServiceClass).inScope("tenant");

// Using ProviderManager
providerManager.register(ServiceIdentifier, "tenant");
```

### Standard Injection Example

```typescript
import { provide, inject } from "@expressots/core";

@provide(UserService)
export class UserService {
    constructor(
        @inject(UserRepository) private userRepo: UserRepository,
        @inject(EmailService) private emailService: EmailService,
    ) {}
}

// Optional injection
@provide(NotificationService)
export class NotificationService {
    constructor(
        @inject(PushService) @optional() private pushService?: PushService,
    ) {}
}

// Named bindings
@provide(DatabaseService)
export class DatabaseService {
    constructor(
        @inject("PrimaryDB") @named("primary") private primaryDb: Database,
        @inject("ReplicaDB") @named("replica") private replicaDb: Database,
    ) {}
}
```
