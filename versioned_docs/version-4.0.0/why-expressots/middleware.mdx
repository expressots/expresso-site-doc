# Middleware Comparison

Compare ExpressoTS's middleware system with other major frameworks.

| **Global Middleware** | ✅ `configureServices()` | ✅ `@Configuration` | ✅ `app.Use()` | ✅ `app.use()` | ✅ `MIDDLEWARE` | ✅ Kernel |

### ExpressoTS Advantage

- ✅ **Most comprehensive middleware system** - 15+ built-in middleware helpers
- ✅ **Middleware Presets** - Pre-configured bundles (`api`, `production`, `development`) - **UNIQUE**
- ✅ **Performance Profiling** - Built-in middleware profiler with metrics - **UNIQUE**
- ✅ **Pipeline Introspection** - Query middleware pipeline at runtime - **UNIQUE**
- ✅ **Zero dependencies** - All middleware helpers included, no external packages needed

### Unique Features

1. **`when()` / `unless()`** - Conditional execution based on request context
2. **`combine()` / `sequence()`** - Semantic middleware grouping
3. **Class references without `new` keyword** - Cleaner API

### Conditional Middleware Example

```typescript
import { when, unless } from "@expressots/adapter-express";

// Execute middleware only when condition is true
@Get("/admin",
    when(req => req.hostname.startsWith("admin."), AdminMiddleware),
    AuthMiddleware
)
async adminHandler() {}

// Execute middleware unless condition is true
@Get("/public",
    unless(req => req.headers.authorization, AuthMiddleware)
)
async publicHandler() {}

// Async conditions are also supported
@Get("/api",
    when(async req => {
        const user = await getUser(req);
        return user.isAdmin;
    }, AdminMiddleware)
)
async apiHandler() {}
```

### Middleware Composition Example

```typescript
import { combine, sequence } from "@expressots/adapter-express";

// Combine multiple middleware into a reusable group
@Get("/api", combine(AuthMiddleware, LoggingMiddleware, RateLimitMiddleware))
async apiHandler() {}

// Sequence middleware where each depends on the previous
@Get("/data", sequence(ValidateMiddleware, TransformMiddleware, ProcessMiddleware))
async dataHandler() {}
```

### Middleware Presets

Pre-configured middleware bundles for common scenarios:

- **`api`** - CORS, Helmet, Body Parser, Compression, Rate Limiting
- **`production`** - All security middleware, compression, rate limiting  
- **`development`** - Development-friendly configuration with detailed logging

```typescript
export class App extends AppExpress {
    async configureServices(): Promise<void> {
        this.Middleware.usePreset("api");
    }
}
```
