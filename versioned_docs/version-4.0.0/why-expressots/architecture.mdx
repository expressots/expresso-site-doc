# Architecture Comparison

Compare ExpressoTS's application architecture with other major frameworks.

## Application Factory/Bootstrap

| Feature | ExpressoTS | Spring Boot | ASP.NET Core | NestJS | Django | Laravel |
|---------|-----------|-------------|--------------|--------|--------|---------|
| **Bootstrap Function** | ✅ **`bootstrap()`** - Zero-config startup | ✅ `SpringApplication.run()` | ✅ `WebApplication.Create()` | ✅ `NestFactory.create()` | ✅ `django.setup()` | ✅ `Application::make()` |
| **Application Factory** | ✅ `AppFactory.create()` (internal/advanced) | ⚠️ Internal | ⚠️ Internal | ⚠️ Internal | ⚠️ Internal | ⚠️ Internal |
| **Environment Detection** | ✅ **UNIQUE** - CI/CD auto-detection | ⚠️ Manual | ⚠️ Manual | ⚠️ Manual | ⚠️ Manual | ⚠️ Manual |
| **Smart .env Loading** | ✅ **UNIQUE** - Opt-in with auto-creation | ⚠️ Via Spring Boot Config | ⚠️ Via configuration | ⚠️ Via @nestjs/config | ⚠️ Via django-env | ⚠️ Via .env file |
| **Port Resolution** | ✅ **Priority chain** - options \> env \> 3000 | ✅ Via application.properties | ✅ Via configuration | ⚠️ Manual | ⚠️ Manual | ⚠️ Manual |
| **Package.json Metadata** | ✅ **Auto-extract** - name & version | ⚠️ Via Maven/Gradle | ⚠️ Via .csproj | ⚠️ Via package.json | ⚠️ Via setup.py | ⚠️ Via composer.json |
| **Application Builder** | ✅ `IWebServerBuilder` | ✅ `ApplicationContext` | ✅ `WebApplicationBuilder` | ✅ `INestApplication` | ✅ `WSGI/ASGI` | ✅ Kernel |
| **Graceful Shutdown** | ✅ **ADVANCED** - All signals + CLI | ✅ Basic | ✅ `app.StopAsync()` | ✅ `app.close()` | ⚠️ Manual | ⚠️ Manual |
| **Application Lifecycle** | ✅ **UNIQUE** - Auto-discovery | ✅ `@PostConstruct` | ✅ `IHostedService` | ✅ Module hooks | ⚠️ Signals | ✅ Service Providers |

### ExpressoTS Advantage

- ✅ **Zero-config bootstrap** - `bootstrap(App)` handles everything (env, port, metadata, startup)
- ✅ **Smart environment loading** - Opt-in .env file loading with auto-creation and CI/CD detection
- ✅ **Unique auto-discovery lifecycle** with `IBootstrap` and `IShutdown` interfaces
- ✅ **Signal-aware shutdown** - providers receive the actual signal (SIGTERM, SIGINT, etc.)
- ✅ **CI/CD auto-detection** - Automatically detects CI environment and adjusts behavior
- ✅ **Package.json integration** - Auto-extracts app name and version

### Bootstrap Example

```typescript
// Recommended: Zero-config bootstrap (handles everything)
import { bootstrap } from "@expressots/core";

await bootstrap(App);

// With custom options
await bootstrap(App, {
    port: 4000,
    appName: "My API",
    appVersion: "2.0.0",
    currentEnvironment: "production",
    envFileConfig: {
        files: {
            development: ".env.dev",
            production: ".env.prod",
        },
        required: ["DATABASE_URL", "API_KEY"],
        autoCreateTemplate: true,
    },
});
```

### Bootstrap Features

- **8 Critical Startup Phases**: Environment detection → Smart .env loading → Port determination → Package.json metadata → DI container init → Environment injection → API version detection → Server startup
- **CI/CD Auto-Detection**: Detects GitHub Actions, GitLab CI, Jenkins, CircleCI, Travis, Buildkite, Azure DevOps, Bamboo, TeamCity
- **Smart .env Loading**: Opt-in behavior, auto-creates templates, validates required variables, platform-specific error hints
- **Port Resolution**: Priority chain (options.port \> process.env.PORT \> 3000), supports port 0 for testing
- **Package.json Integration**: Auto-extracts name and version, cached for performance
- **Performance**: ~8-25ms typical startup time (optimized with caching)

## Modular Architecture

| Feature | ExpressoTS | Spring Boot | ASP.NET Core | NestJS | Django | Laravel |
|---------|-----------|-------------|--------------|--------|--------|---------|
| **Module System** | ✅ `ContainerModule` + `CreateModule()` | ✅ `@Configuration` | ✅ Service registration | ✅ `@Module()` | ✅ Apps | ✅ Service Providers |
| **Feature Modules** | ✅ Auto-binding via `@provide()` | ✅ Component scan | ✅ Service scoping | ✅ Declarative | ✅ Installed apps | ✅ Auto-discovery |
| **Dynamic Modules** | ✅ Factory functions (simpler) | ✅ `@Bean` | ✅ Configuration-based | ✅ `DynamicModule` | ⚠️ Manual | ✅ Deferred providers |
| **Module Composition** | ✅ **UNIQUE** - `combineModules()` | ⚠️ Import | ⚠️ Manual | ⚠️ Via arrays | ⚠️ Manual | ⚠️ Manual |

### ExpressoTS Advantage

- ✅ **Simplest module system** - Auto-binding eliminates explicit module declarations
- ✅ **Composition utilities** - `CreateModule()`, `createModule()`, `combineModules()`
- ✅ **Less boilerplate** - No need for explicit imports/exports like NestJS

### Module Example

```typescript
// ExpressoTS: Simple, auto-binding approach
import { CreateModule } from "@expressots/core";

// Controllers are organized via CreateModule
export const UserModule = CreateModule([
    UserController,
    ProfileController,
]);

// Services are auto-discovered via @provide()
@provide(UserService)
export class UserService {
    // Automatically registered in container
}
```
